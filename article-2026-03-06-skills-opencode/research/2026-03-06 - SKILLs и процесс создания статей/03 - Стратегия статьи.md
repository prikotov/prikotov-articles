## 3. Позиционирование

### Уникальность статьи

**Чем отличается от других:**

1. **Практический кейс** — не теория, а реальный эксперимент
2. **Data-driven** — основано на данных, не на мнении
3. **Открытые skills** — можно сразу использовать
4. **Честный тон** — "результат не идеален, но процесс работает"
5. **Процессный подход** — фокус на улучшаемом процессе, а не на идеальном результате

### Ценность для читателя

| Ценность | Что получит читатель |
|----------|---------------------|
| **Практическая** | Готовые skills для аналитики блога |
| **Методологическая** | Понимание как выстроить процесс |
| **Инструментальная** | Знакомство с OpenCode + GLM-5 |
| **Идеологическая** | Процесс > результат, улучшение > перфекционизм |
| **Эмоциональная** | "У меня тоже получится!" |

### CTA (Call-to-Action)

**Основной CTA:** Попробовать skills для своего проекта

**Варианты:**

1. **Мягкий (заключение):**
   - "Скилы открыты, берите, адаптируйте под себя"
   - "Попросите агента настроить их под ваш проект"

2. **Средний (в тексте):**
   - "Я описал процесс в AGENTS.md, можете взять за основу"
   - "Skills на GitHub — ссылка в конце статьи"

3. **Прямой (если уместно):**
   - "Если нужен AI-агент для вашего проекта — task.ai-aid.pro"

**Выбор:** Мягкий + средний. Не продаём, а делимся.

## 4. Связь с task.ai-aid.pro

### Контекст упоминания

| Место | Как упомянуть | Тон |
|-------|---------------|-----|
| **Введение** | "Мой блог помогает привлекать аудиторию для проекта task.ai-aid.pro" | Факт, не продажа |
| **Проблема** | "Я трачу много времени на аналитику, чтобы понять что заходит аудитории" | Честность |
| **Решение** | "Я выстроил процесс для кода (task-agents-playbook), теперь применяю к статьям" | Экспертиза |
| **Skills** | "Skills открытые, можно адаптировать под любой проект" | Польза |
| **Заключение** | "Такой подход можно применить к любому процессу" | Универсальность |

### Как показать экспертизу в AI-агентах

1. **Реальный опыт** — "Я написал skills за 3 дня между перерывами"
2. **Практические примеры** — код, команды, результаты
3. **Честные выводы** — "Результат не идеален, но процесс работает"
4. **Открытость** — skills на GitHub, AGENTS.md опубликован
5. **Процессное мышление** — фокус на улучшении, а не на результате

### Что НЕ делать

- ❌ "Вам нужен task.ai-aid.pro"
- ❌ "Лучший способ — использовать task.ai-aid.pro"
- ❌ "Я эксперт в AI-агентах"
- ❌ Продажа вместо истории

## 5. Формат статьи

### Тип статьи

**Кейс + Туториал**

- Кейс: личный эксперимент с процессом
- Туториал: как использовать skills
- Обзор: OpenCode + GLM-5 + skills

### Длина

**Средняя-длинная (3000-4000 слов)**

- Введение: 200 слов
- Проблема: 300 слов
- Эксперимент: 500 слов
- Skills: 800 слов (с примерами кода)
- Процесс: 600 слов
- Результат: 400 слов
- Выводы: 200 слов

### Структура

1. **Введение** — хук, проблема, решение
2. **Проблема** — почему я начал этот эксперимент
3. **Эксперимент** — OpenCode + GLM-5 + skills
4. **Skills** — какие skills, как работают, примеры
5. **Процесс** — AGENTS.md, этапы, workflow
6. **Результат** — что получилось, что можно улучшить
7. **Выводы** — главный инсайт, CTA

### Элементы

| Элемент | Количество | Цель |
|---------|------------|------|
| **Mermaid диаграммы** | 2-3 | Визуализация процесса |
| **Примеры кода** | 3-5 | Практическая польза |
| **Ссылки** | 5-8 | GitHub, блог, task.ai-aid.pro |
| **Цитаты** | 1-2 | Акцент на ключевых мыслях |
| **Таблицы** | 3-5 | Структурирование данных |

### Mermaid диаграммы

1. **Процесс создания статьи** (flowchart)
   - Сбор данных → Анализ → Стратегия → Драфт → Публикация

2. **Архитектура skills** (diagram)
   - OpenCode → Skills → APIs → Reports

3. **Цикл улучшения** (flowchart)
   - Процесс → Результат → Анализ → Улучшение → Процесс

## 6. Tone of Voice

### Голос автора

**Честный практик, не эксперт**

- ✅ "У меня нет опыта в продвижении"
- ✅ "Я решил попробовать..."
- ✅ "Оказалось, что..."
- ✅ "Результат не идеален, но..."
- ❌ "Как эксперт, я рекомендую..."
- ❌ "Лучший способ — ..."
- ❌ "Вам нужно..."

### Стиль

| Аспект | Подход |
|--------|--------|
| **Язык** | Простой, без жаргона (или с объяснением) |
| **Темп** | Динамичный, короткие абзацы |
| **Структура** | Подзаголовки каждые 300-500 слов |
| **Примеры** | Реальный код, реальные данные |
| **Честность** | Признание ошибок, неидеальности |

### Эмоциональный тон

- Заинтересованность: "Оказалось, что skills — это мощно!"
- Честность: "Результат не идеален"
- Практичность: "Вот код, который работает"
- Открытость: "Берите skills, адаптируйте"
